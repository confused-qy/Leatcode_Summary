# 贪心算法（Greedy Algorithm）

贪心算法的核心思想是：

> 每一步都选择当前看起来最优的方案，希望最终得到全局最优解。

与动态规划不同，贪心算法不会回头修改之前的决策，因此通常实现简单、效率较高。

---

# 适用场景

贪心算法通常适用于：

- 当前最优选择不会影响后续决策
- 局部最优能够推出全局最优
- 问题具有贪心选择性质（Greedy Choice Property）

如果每一步的最优选择都能保证最终答案最优，通常可以考虑贪心。

---

# 常见特征

看到以下关键词时，可以优先考虑贪心：

- 最大化 / 最小化某个指标
- 最少次数
- 最多数量
- 区间选择
- 资源分配
- 排序后依次选择

很多贪心题的第一步都是：

```cpp
sort(nums.begin(), nums.end());
```

然后按照某种规则依次选择。

---

# 判断是否能用贪心

如果想到一种“当前最好”的策略，可以先问：

```text
当前选择会不会导致后面更差？
```

如果不会，并且能够证明局部最优最终得到全局最优，则可以使用贪心。

否则通常需要考虑：

```text
动态规划（DP）
回溯（Backtracking）
搜索（DFS/BFS）
```

---

# 一句话总结

```text
贪心算法 = 每一步都做当前最优选择，并且不回头修改决策。
```
