# 回溯（Backtracking）是什么

回溯可以理解成：“不断尝试所有可能，如果发现当前方案不行，就退回来换一种。” 它本质上是一种搜索方法。比如在全排列问题里，我们先选一个数字，再继续选下一个数字，如果已经形成一个完整答案就保存；然后再“退回来”，撤销刚才的选择，去尝试别的可能。这个“退回来重新尝试”的过程，就是回溯。

回溯通常会配合递归使用，因为递归非常适合“一层一层往下搜索”。例如，第一层决定第一个位置放什么，第二层决定第二个位置放什么，第三层决定第三个位置放什么。于是代码经常会写成：先做一个选择，然后递归进入下一层，最后再撤销这个选择。像 `push_back()` 和 `pop_back()` 这种“加入”和“撤销”的操作，就是回溯的核心。

递归和回溯虽然长得很像，但重点不同。递归本质上只是“函数调用自己”，通常用于把一个大问题拆成更小的问题，例如阶乘、二叉树、Fibonacci 等。而回溯更强调“尝试所有可能”，并且在尝试结束后，需要“恢复现场”。所以可以把回溯理解成：

```text
回溯 = 递归 + 撤销操作
```

经典模版：
``` cpp
path.push_back(x);

dfs();

path.pop_back();
```
